Some very useful open-source libraries in C and C++

  • Boost: One of the very powerful open-source libraries in C++. It consists of very efficient libraries for Arrays, Algorithms, Geometry, Graphs, Math functions, Timers etc.
  • Eigen: A very efficient Matrix library built purely based on Expression Templates. It does not require compilation.
  • PETSc: A parallel Matrix Algebra library with numerous parallel iterative solvers and support for many third-party solver libraries.
  • Intel MKL: Matrix library from Intel. (You have to purchase it!)
  • CGAL: A powerful library for Geometry processing algorithms. CGAL's algorithms are so much more robust and efficient than their counterparts in VTK. Between VTK and CGAL, always use CGAL.
  • VTK: Along with ParaView, VTK is one of the best open-source libraries (in my opinion) for data visualisation for FEM/CFD applications. VTK has a large collection of algorithms for geometry processing when compared with CGAL, however, VTK algorithms are not as robust and efficient as those in CGAL.
  • Sparse direct solvers (also for FORTRAN)
    • PARDISO - parallelisation with OpenMP
    • SUPERLU - parallelisation with OpenMP (SuperLU_MT) as well as OpenMPI (SuperLU_DIST)
    • MUMPS - parallelisation with OpenMPI
    • UMFPACK - Used in MATLAB
      • There are many others, for example, SPOOLES, TAUCS etc. But, they are not as good the ones listed above.

Computer Graphics and Computer Aided Design

  • CGAL: A powerful library for Geometry processing algorithms.
  • FreeCAD: Opensource 3D parametric modeller. It also includes simple mesh generator along with a basic FE solver.

FEM/CFD

  • Gmsh: Gmsh is a free 3D finite element mesh generator with a built-in CAD engine and post-processor.
  • HyperMesh: Commercial software suit for pre-processing for FEA/CFD applications. The best mesh generation software available for large, complex CAD geometries. Supports mesh export for various commercial FEA/CFD software packages, for example, ANSYS, Abaqus, Nastran, etc.
    Note: HyperMesh has a steep initial learning curve but it is worth the effort. Once you know HyperMesh, meshing is never going to be an ordeal in your life. HyperMesh is so much more user-friendly than Gmsh.

Visualisation

  • ParaView: Built on top of VTK library, Paraview is a free software tool which supports a lot of input formats and consists of several functions for post-processing FEA/CFD results, including the support for creating animations. Paraview also has support for parallel data visualisation. Models with billions of cells have been post-processed using parallel Paraview server.
  • Matplotlib: Python based tool for visualisation of data. Especially good for plotting graphs and contours. My preferred choice over GNUplot. Supports maths symbols using LaTex format.
